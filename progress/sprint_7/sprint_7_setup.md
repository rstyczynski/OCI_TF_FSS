### Assessment

PBI-019 refactors the stack module variable interface so that mount targets and filesystems become independent, first-class map inputs. Each filesystem entry carries a nested `exports` map; each export references a mount target by stable key. This models OCI's actual M:N relationship — one mount target can serve exports from multiple filesystems, and one filesystem can be exported via multiple mount targets.

The new module is created at `terraform/modules/fss_sprint7_stack`. The existing `fss_sprint5_stack` module is preserved unchanged as a compatibility baseline. PBI-019 supersedes PBI-015 (per-filesystem parameter overrides), so PBI-015 is considered closed by this sprint.

**Key implementation challenge — `for_each` flattening:**
The nested `exports` map inside `filesystems` prevents direct iteration. The canonical Terraform pattern is to produce a flattened collection of `(filesystem_key, export_key)` pairs using `flatten` and a local value, then use a composite string key such as `"${fs_key}__${export_key}"` as the `for_each` key. These keys must be stable so that adding or removing one filesystem/export entry does not force replacement of unrelated resources.

**Variables introduced by the sprint:**

```hcl
variable "mount_targets" {
  type = map(object({
    display_name   = optional(string)
    hostname_label = optional(string)
    nsg_ids        = optional(list(string))
    freeform_tags  = optional(map(string), {})
    defined_tags   = optional(map(string), {})
  }))
  default = {}
}

variable "filesystems" {
  type = map(object({
    display_name  = string
    freeform_tags = optional(map(string), {})
    defined_tags  = optional(map(string), {})
    exports = optional(map(object({
      mount_target_key               = string
      path                           = string
      source                         = optional(string, null)
      access                         = optional(string, "READ_WRITE")
      allowed_auth                   = optional(list(string), ["SYS"])
      identity_squash                = optional(string, "ROOT")
      anonymous_uid                  = optional(number, 65534)
      anonymous_gid                  = optional(number, 65534)
      is_anonymous_access_allowed    = optional(bool, false)
      require_privileged_source_port = optional(bool, false)
    })), {})
  }))
  default = {}
}
```

**Shared defaults retained** (`compartment_ocid`, `availability_domain`, `subnet_ocid`, `kms_key_id`, `default_source_cidr`) — same mandatory/optional split as the Sprint 5 stack.

**Outputs:** the composite `filesystems` output is replaced by a `mount_targets` output (per mount target) and a `filesystems` output (per filesystem, with nested export summaries keyed by export key). Atomic outputs (`mount_target_ocids`, `export_paths`, `nfs_mount_sources`) follow the Sprint 5 TF rules for composite + atomic pairing.

**Regression scope:** `Regression: none` as stated in PLAN.md — the Sprint 5 stack tests are not re-executed because the Sprint 5 module is unchanged.
